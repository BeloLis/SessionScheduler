# SessionSchedulerGUI

Настольное Windows-приложение (C++17, MFC) для автоматического составления расписания экзаменационной сессии вуза из Excel-файла: зачётов, курсовых работ, зачётов с оценкой, курсовых проектов и экзаменов с консультациями.

Проект вырос из выпускной квалификационной работы бакалавра «Автоматизация составления расписания сессии» (СПбПУ, Высшая школа программной инженерии, 2025) и с тех пор дорабатывается как pet-проект: часть ограничений и ошибок, описанных в самой работе как направления для улучшения, уже устранена в этом репозитории.

## Возможности

- Чтение всех исходных данных сессии из одного Excel-файла (`.xlsx`): дни сессии, здания и аудитории, потоки студентов, зачёты/курсовые работы, зачёты с оценкой/курсовые проекты, экзамены.
- Учёт ручных фиксированных назначений (лист «Итог» во входном файле): если для конкретной группы, предмета и преподавателя аттестация уже проведена или жёстко закреплена на определённую дату и время, алгоритм не переставляет её, но обязательно учитывает как занятый слот при распределении остальных аттестаций (аудитория/преподаватель/группа считаются занятыми).
- Автоматическое планирование консультаций перед экзаменом с учётом дней подготовки, кроме случаев, когда консультация уже зафиксирована вручную в листе «Итог» - тогда она не переназначается повторно.
- Проверка нормативных ограничений: не более одной аттестации у преподавателя/группы/аудитории одновременно, минимальные интервалы между аттестациями группы, дневная нагрузка преподавателя, предпочтения по датам и типам аудиторий (обычная/компьютерная/дистанционная), исключённые даты и праздники.
- Приоритизация задач в очереди планирования (сильно загруженные преподаватели и более «жёсткие» по ограничениям аттестации распределяются первыми).
- Формирование результата в двух листах итогового Excel-файла: «Распределённые_аттестации» (включает как ранее зафиксированные вручную записи, так и вновь распределённые алгоритмом) и «Несоставленные_аттестации» (с указанием причины, по которой аттестацию не удалось поставить).
- Простой графический интерфейс на MFC: выбор входного и выходного файла, запуск расчёта в отдельном потоке, лог хода выполнения.
- Подробное логирование чтения/записи Excel (`excel_log.txt`) и работы планировщика (`scheduler.log`) для разбора причин конкретных решений алгоритма.

## Как это работает

В основе - жадный алгоритм (greedy): все задачи (зачёты, курсовые, зачёты с оценкой, курсовые проекты, экзамены) сортируются по приоритету и последовательно получают первый подходящий по всем ограничениям слот (день, время, аудитория). Для экзаменов дополнительно подбирается дата консультации в рамках допустимых дней подготовки перед экзаменом.

Фиксированные записи из листа «Итог» читаются один раз при старте, помечаются как неизменяемые и участвуют во всех проверках занятости наравне с уже распределёнными в текущем прогоне задачами - то есть новый экзамен не может быть поставлен в ту же аудиторию/тому же преподавателю/той же группе, если время пересекается с ранее зафиксированной вручную записью.

## Формат входного файла

Входной `.xlsx`-файл должен содержать следующие листы:

| Лист | Назначение |
| --- | --- |
| `Дни сессии` | Календарь сессии: рабочие дни, выходные, праздники |
| `Поток` | Состав учебных потоков (какие группы входят в поток) |
| `Аудитории` | Здания, номера аудиторий, вместимость, тип (обычная/компьютерная), доступное время |
| `З_КР` | Зачёты и курсовые работы |
| `ЗаО_КП` | Зачёты с оценкой и курсовые проекты |
| `Экзамен` | Экзамены (с указанием часов на подготовку и необходимости консультации) |
| `Итог` | *Необязательный.* Ручные фиксированные назначения - конкретные группа/предмет/преподаватель/аудитория/дата/время, которые нельзя двигать алгоритму |

Лист «Итог» - это вход, а не результат работы программы: он заполняется составителем расписания вручную до запуска, если для каких-то аттестаций дата и аудитория уже согласованы и не подлежат изменению. Программа никогда не записывает и не изменяет этот лист.

## Формат выходного файла

Результат сохраняется в отдельный `.xlsx`-файл (исходный файл при этом не изменяется) и содержит:

- **Распределённые_аттестации** - итоговое расписание: группа, дисциплина, аудитория, дата, время начала и окончания, тип аттестации, преподаватель. Включает как записи, перенесённые без изменений из листа «Итог», так и новые, поставленные алгоритмом.
- **Несоставленные_аттестации** - задачи, для которых не нашлось подходящего слота, с указанием причины (например, нет свободной аудитории, преподаватель перегружен, исчерпан диапазон дат сессии).

## Архитектура проекта

```
SessionSchedulerGUI/
├── SessionSchedulerGUI.cpp / .h        # Точка входа, инициализация MFC-приложения
├── SessionSchedulerGUIDlg.cpp / .h     # Главное окно: выбор файлов, запуск расчёта, лог в UI
├── ExcelReader.cpp / .h                # Чтение всех листов входного файла и запись результата (xlnt, Pimpl)
├── Scheduler.cpp / .h                  # Жадный алгоритм распределения аттестаций и консультаций
├── SessionDay / Building / StudentFlow # Модели данных: дни сессии, здания и аудитории, потоки
├── Exam / GradedExam / FinalExam       # Модели трёх типов аттестаций (З/КР, ЗаО/КП, Экзамен)
├── Teacher / TimePreference            # Модели преподавателя и временных предпочтений
├── ScheduleEntry.h                     # Единица итогового расписания (группа, аудитория, дата, время, тип, преподаватель)
├── DateUtils.cpp / .h                  # Разбор дат, дни недели, праздники, соседние даты
└── Utils.cpp / .h                      # Вспомогательное форматирование (например, время в HH:MM)
```

Работа с Excel вынесена в отдельный класс `ExcelReader` по идиоме Pimpl (реализация на `xlnt` скрыта во внутреннем классе `Impl`), а вся логика планирования - в `Scheduler`, не зависящий напрямую от формата Excel.

## Требования для сборки

- Windows + Visual Studio 2022 (или новее) с компонентом **«Разработка классических приложений на C++»**
- Git
- [vcpkg](https://github.com/microsoft/vcpkg) для подключения библиотеки [xlnt](https://github.com/tfussell/xlnt) (чтение/запись `.xlsx`)

## Как собрать

1. Установите vcpkg (если ещё не установлен):

   ```bash
   git clone https://github.com/microsoft/vcpkg
   cd vcpkg
   bootstrap-vcpkg.bat
   vcpkg integrate install
   ```

2. Склонируйте этот репозиторий:

   ```bash
   git clone <ссылка на ваш репозиторий>
   ```

3. Откройте `SessionSchedulerGUI.sln` в Visual Studio.
4. Выберите конфигурацию **Release** и платформу **x64**.
5. Соберите: **Сборка → Перестроить решение**.

   При первой сборке vcpkg автоматически скачает и соберёт нужную библиотеку (`xlnt`) - это может занять 15–25 минут. При последующих сборках будет быстро.

6. Готовый файл появится в `x64\Release\SessionSchedulerGUI.exe`.

## Как использовать

1. Запустите `SessionSchedulerGUI.exe`.
2. Укажите путь к входному Excel-файлу с данными сессии (см. [формат входного файла](#формат-входного-файла)).
3. Укажите путь для выходного файла с результатом.
4. Нажмите «Запуск» - расчёт выполняется в фоновом потоке, ход выполнения отображается в логе окна.
5. По завершении откройте выходной файл: листы «Распределённые_аттестации» и «Несоставленные_аттестации».
6. Если часть аттестаций не удалось распределить - посмотрите причины на листе «Несоставленные_аттестации» и при необходимости смягчите ограничения (например, исключённые даты преподавателя) во входном файле, затем запустите расчёт повторно.

## Известные ограничения

- Жадный алгоритм не гарантирует глобально оптимального решения: при очень высокой нагрузке на одного преподавателя часть его аттестаций может остаться нераспределённой (подробный разбор — в тексте ВКР, глава 4).
- Равномерность распределения нагрузки по дням и аудиториям - не строго оптимизационная задача, а побочный эффект жадного подхода.
- Интерфейс минимальный (выбор файлов и лог), без предпросмотра и ручного редактирования расписания в самом приложении.

## История / что было исправлено после сдачи ВКР

В процессе доработки после защиты ВКР были найдены и устранены следующие проблемы:

- **Некорректная обработка путей с кириллицей.** Проверка существования входного/выходного файла интерпретировала UTF‑8-путь как ANSI-кодовую страницу, из‑за чего файл с русскими символами в пути считался «не найденным», а результат сохранялся в пустую книгу без исходных листов.
- **Фиксированные записи из листа «Итог» не блокировали слоты.** Записи читались, но не учитывались при проверке занятости аудиторий/преподавателей/групп, из-за чего алгоритм мог поставить новую аттестацию поверх уже зафиксированной вручную.
- **Некорректный разбор времени в листе «Итог».** Числовые ячейки времени Excel (доля суток) делились на 24 повторно, что давало время около полуночи вместо реального.
- **Повторное планирование уже зафиксированной консультации.** Если консультация перед экзаменом уже была явно указана в листе «Итог», система всё равно пыталась запланировать её заново.
